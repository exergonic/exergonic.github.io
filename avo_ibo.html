avo_ibo

GitHub · BSD 3-Clause · 2025–2026

Here is the thing about Intrinsic Bond Orbitals (IBOs): they give you a beautifully clear picture of chemical bonding — σ bonds, π bonds, lone pairs, and core orbitals — straight from quantum chemistry. You can see the bent bonds in cyclopropane, the π cloud in ethene, the lone pairs on water. They are exactly the kind of orbital you drew on a chalkboard as a student, except these are real, computed from first principles.

But up until now, getting them has been a hassle. IboView is the reference implementation, but it is a standalone tool with its own workflow — not something you can just drop into your existing molecular editor. You would run a calculation, export the results, open them somewhere else, coax the isosurfaces to render… it was doable, but it was not easy.

avo_ibo changes that. It is an Avogadro 2 plugin (with a standalone CLI) that runs the full IBO pipeline using Psi4 as the quantum chemistry backend. You install it, open a molecule, click Extensions → Intrinsic Bond Orbitals → Compute IBOs, and your orbitals appear in the Molecular Orbitals panel — ready to click, rotate, and explore. No file juggling. No separate viewer. Just compute and visualize.

What You Can Do With It

  • Compute IBOs for any neutral closed-shell molecule using HF or DFT — water, methane, ethene, transition states, whatever you are working on
  • Browse your orbitals in Avogadro’s Molecular Orbitals panel — double-click any orbital to toggle its isosurface, rotate the molecule, inspect the shape
  • View both occupied IBOs (σ bonds, π bonds, lone pairs, core orbitals) and valence-virtual IBOs (σ* and π* anti-bonding orbitals)
  • Get a full analysis table with occupancy, energy, bond type, atomic composition, and s/p/d hybridization for every orbital
  • Run from the command line with python -m avogadro_ibo molecule.xyz for headless computation
  • Customize the basis set and method (cc-pVDZ, cc-pVTZ, B3LYP, etc.) through the options

Quick Start

Requires pixi.

git clone https://github.com/exergonic/avo_ibo.git
cd avo_ibo
pixi install
pixi run python -m pip install -e .

Standalone CLI

pixi run python -m avogadro_ibo molecule.xyz

Reads an XYZ file, runs the full IBO pipeline, and writes results to calcs/last/ (ibo.molden, canonical.molden, ibos.txt, psi4.log). Neutral singlet only.

Avogadro 2 plugin

Create a symlink so Avogadro discovers the plugin:
(requires Administrator or Developer Mode) 

New-Item -ItemType SymbolicLink -Path "$env:LOCALAPPDATA\OpenChemistry\Avogadro\plugins\avo_ibo" `
  -Target "C:\path\to\avo_ibo"

Then in Avogadro: Extensions → Intrinsic Bond Orbitals → Compute IBOs. The computed IBOs appear in the Molecular Orbitals panel where you can double-click any orbital to toggle its isosurface.

How It Works (At a Glance)

You give it a molecule. It spins up Psi4 for a quantum chemistry calculation, builds the Intrinsic Atomic Orbital basis that exactly spans your occupied molecular orbitals, then localizes them with the Pipek–Mezey procedure. The result is a set of orbitals that look like what a chemist would draw: bonds localized between atoms, lone pairs on single atoms, core orbitals deep in the electron cloud. Virtual orbitals (σ*, π*) are constructed via an SVD projection and localized the same way.

Everything runs in a single Python process — no subprocess calls, no intermediate files to manage. The plugin talks to Avogadro over stdin and stdout using JSON, and a debug log captures every input and output for troubleshooting.

A full mathematical derivation of every step is in mathematics.md for those who want the equations.

For the Professionals

The sections below cover the technical details: the IAO/2014 algorithm, Pipek–Mezey localization with on-atom degeneracy resolution, the SVD-based valence-virtual construction, tested molecules, the donor–acceptor analysis limitation, and the build architecture.

Pipeline Overview

Geometry → Psi4 HF/cc-pVDZ → C_occ (occupied MO coefficients)
    ↓
C_IAO, C_IAO_occ = _build_iao_basis(S_full, S12, S_min, C_occ)
    ↓
_localize_ibos(C_IAO_occ)      ← PM p=2 → p=4 warm-start
    ↓
_resolve_on_atom_mixing(C_IAO_occ)  ← Fock diag per same-atom group
    ↓
Valence-virtual: SVD(S_IbVir) → U_val → _localize_ibos(U_val)
    ↓
Combine, sort by energy → Molden output → analysis table

The IAO construction follows Appendix C of the Knizia 2013 paper, matching IboView’s MakeIaoBasisNew. A minimal basis (STO-3G) is projected into the full AO space, and the resulting Intrinsic Atomic Orbitals are symmetrically orthogonalized. These IAOs exactly span the occupied space — every occupied MO can be expressed as a linear combination of IAOs with zero loss.

The Pipek–Mezey localization (p=4, convergence 1e-12, max 2048 iterations) uses a warm-start strategy: first solve with the convex p=2 functional, then refine with p=4. This avoids shallow local minima that can trap a cold p=4 start. The Jacobi sweep algorithm rotates orbital pairs to maximize atomic population concentration, yielding chemically intuitive σ bonds, π bonds, lone pairs, and core orbitals.

On-Atom Degeneracy Resolution

The PM functional depends only on atomic populations, so orbitals that both sit on the same atom with DOM ≈ 1 (e.g. O 2s and the oxygen lone pair in water) can mix arbitrarily without changing the localization measure. After PM, a post-processing step diagonalizes the Fock matrix within each same-atom, high-DOM subspace, restoring the aufbau ordering (s-rich lowest, p-rich highest).

A Note on Donor–Acceptor Analysis

Unlike NBOs, the IAO basis exactly spans the occupied space — it is a lossless representation of the occupied MOs. This exactness is a mathematical identity (proved in the mathematics document), not a numerical approximation. As a consequence, the occupied–virtual off-diagonal block of the Fock matrix in the IAO basis is identically zero, making NBO-style E(2) donor–acceptor analysis structurally impossible. This is a feature of the IBO design: IAOs give clean, chemically intuitive orbital shapes for visualization, but they cannot report hyperconjugation energies.

Architecture

The plugin is structured as a standard Avogadro 2 Pixi Python script plugin. It uses uv_build (not hatchling — the bundled pixi v0.66.0 does not support hatchling) and produces a lock file v6 for compatibility with Avogadro’s pixi. All computation runs in-process (single Python process, no subprocess or file-based IPC to Psi4), with careful logging redirect to avoid stderr contamination of the JSON output that Avogadro parses.

For implementers, a detailed tutorial on building Avogadro 2 Pixi plugins is in tutorial.md (758 lines), covering the architecture, algorithm walkthrough, and every gotcha encountered during development.
